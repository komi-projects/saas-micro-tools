📊 Data Analytics

Collect, analyze, and act on data for better decisions.

Why Data Analytics is Critical

In SaaS, intuition is a terrible substitute for data. "I think users want this feature" leads to wasted engineering months. "The data shows 80% of churned users never tried feature X" leads to a focused onboarding fix that cuts churn by 20%.

Data analytics transforms opinions into evidence. It reveals patterns invisible to human observation. It validates assumptions before you invest. And it measures impact after you ship. Companies that make data-driven decisions grow faster, waste less, and build products people actually want.

This guide covers how to collect the right data, analyze it effectively, and build a culture where data drives decisions.

Data Collection

1. Product Analytics
Track what users do in your product. • Events: Page views, clicks, feature usage, errors • User properties: Signup date, plan, industry, role • Session data: Duration, frequency, device, location • Funnels: Signup → Activation → Retention → Revenue • Cohorts: Group users by signup date, source, or behavior • Tools: Mixpanel, Amplitude, Heap, Pendo • Implementation: Event tracking via SDK or API
2. Business Analytics
Track revenue, costs, and financial health. • Revenue: MRR, ARR, expansion, churn, NRR • Customers: Count, segments, lifecycle stage, health score • Sales: Pipeline, conversion rates, deal size, cycle length • Costs: CAC, COGS, burn rate, runway • Tools: ChartMogul, Baremetrics, Stripe analytics, ProfitWell • Data source: CRM, billing system, accounting software
3. Marketing Analytics
Track acquisition and campaign performance. • Traffic: Visitors, sources, channels, campaigns • Conversion: Lead → MQL → SQL → Opportunity → Customer • Attribution: Which touchpoints drive conversions? • Engagement: Email opens, click-through rates, content performance • ROI: Cost per lead, CAC by channel, ROAS • Tools: Google Analytics, HubSpot, Marketo, Google Ads, Facebook Pixel
4. Customer Analytics
Track satisfaction, feedback, and support. • Surveys: NPS, CSAT, CES, product-market fit • Support: Ticket volume, resolution time, satisfaction • Feedback: Feature requests, bug reports, qualitative interviews • Sentiment: Social media, reviews, community discussions • Tools: Typeform, Delighted, Zendesk, Intercom, Brand24
5. Operational Analytics
Track system health and performance. • Uptime: Availability, response times, error rates • Performance: API latency, database queries, page load times • Infrastructure: CPU, memory, disk usage, costs • Security: Login attempts, access patterns, anomalies • Tools: Datadog, New Relic, Grafana, Prometheus, PagerDuty

Analysis Frameworks

1. Cohort Analysis:

• Group users by a shared characteristic (signup date, acquisition channel, plan type) • Track each cohort's behavior over time (retention, revenue, feature adoption) • Compare cohorts: Are newer users retaining better or worse? • Identify trends: What changed in March that improved April's cohort? • Applications: Retention curves, LTV prediction, CAC payback by cohort • Tools: Mixpanel, Amplitude, Excel, SQL

2. Funnel Analysis:

• Map the user journey: Signup → Activation → Engagement → Revenue → Retention • Measure conversion at each step: What % advance from stage to stage? • Identify drop-offs: Where do users get stuck or leave? • Test fixes: A/B test changes at the biggest drop-off points • Segment funnels: Compare funnels by channel, persona, or device • Applications: Onboarding optimization, sales process improvement, feature adoption

3. Segmentation:

• Divide users into meaningful groups for targeted analysis • Demographic: Company size, industry, geography, role • Behavioral: Power users, casual users, at-risk users, champions • Acquisition: Organic, paid, referral, partner • Engagement: Daily active, weekly active, monthly active, dormant • Value: High LTV, low LTV, expansion candidates, churn risks • Applications: Personalized marketing, targeted interventions, product prioritization

4. Correlation and Causation:

• Correlation: "Users who do X are more likely to do Y" • Causation: "Changing X causes Y to change" (requires experimentation) • A/B testing: The gold standard for proving causation • Multivariate analysis: Controlling for confounding variables • Regression: Predicting outcomes based on multiple factors • Applications: Identifying activation events, churn predictors, expansion signals

5. Predictive Analytics:

• Churn prediction: Which customers are likely to cancel? • LTV prediction: How much will a customer generate over their lifetime? • Lead scoring: Which prospects are most likely to convert? • Demand forecasting: How much capacity will we need next quarter? • Applications: Proactive retention, sales prioritization, capacity planning • Tools: Python (scikit-learn, TensorFlow), R, BigQuery ML, custom models

Analytics Tools

Product analytics:

• Mixpanel: Event-based analytics, funnels, cohorts, A/B testing • Amplitude: Behavioral analytics, user journeys, retention analysis • Heap: Auto-capture all events, retroactive analysis • Pendo: Analytics + in-app guidance and surveys • FullStory: Session replay + analytics • LogRocket: Session replay for web apps with error tracking

Business intelligence:

• Tableau: Enterprise BI, powerful visualizations • Looker: Data modeling + visualization, Git-integrated • Power BI: Microsoft ecosystem, affordable • Metabase: Open-source, SQL-based, self-hosted • Google Data Studio (Looker Studio): Free, Google-native • Mode Analytics: SQL + Python/R for advanced analysis

Data infrastructure:

• Snowflake: Cloud data warehouse, scalable • BigQuery: Google Cloud data warehouse, serverless • Redshift: AWS data warehouse • dbt: Data transformation, SQL-based modeling • Fivetran / Stitch: Data pipeline automation (ETL) • Airbyte: Open-source data integration

Building a Data Culture

1. Data accessibility:

• Democratize data: Everyone should access metrics relevant to their role • Self-serve: Tools and training so teams can answer their own questions • Dashboards: Real-time visibility into key metrics for every team • Documentation: Clear definitions, sources, and caveats for each metric • Data requests: Process for requesting new data or analysis

2. Decision-making with data:

• Hypothesis-driven: "We believe X. If true, we'll see Y in the data." • Pre-mortems: "What would the data look like if this feature failed?" • Post-launch analysis: "What happened after we shipped? Did it work?" • Counter-arguments: "What data would prove us wrong?" (avoid confirmation bias) • No data = no decision: If you can't measure it, don't prioritize it

3. Data quality:

• Ownership: Each metric has an owner who validates accuracy • Testing: Data pipelines should have automated tests for accuracy • Monitoring: Alerts when data freshness or quality degrades • Documentation: Data dictionary, schema documentation, lineage tracking • Governance: Who can access what data? How is sensitive data handled?

4. Analytics team:

• Data analysts: Answer business questions, build reports, support teams • Data engineers: Build and maintain data pipelines and infrastructure • Data scientists: Predictive models, advanced statistics, ML • Analytics manager: Prioritize requests, maintain data quality, evangelize data culture • Embedded analysts: One analyst per major team (product, marketing, sales)

Analytics Mistakes

1. Analysis paralysis: Collecting data forever without acting. Data is only valuable if it drives decisions. Set deadlines for analysis and commit to action.

2. Vanity metrics: Tracking "total signups" without conversion rate. "Total revenue" without churn. "Page views" without engagement. Metrics should drive action, not ego.

3. Correlation = causation: "Users who do X retain better" doesn't mean X causes retention. Maybe retained users naturally do X. Test with experiments before building.

4. Ignoring qualitative data: Numbers tell you what. Interviews tell you why. Both are necessary. A churned user's interview reveals what cohort analysis never will.

5. Data silos: Marketing has their data, product has theirs, finance has theirs. Nobody sees the full picture. Centralize data and create cross-functional dashboards.

6. No baseline: "We improved activation by 20%!" Great — from what baseline? And what was the control? Always measure against a baseline or control group.
