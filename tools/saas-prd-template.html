Feature Name
Smart Dunning Dashboard
Owner
Jane Doe, Product Manager
Status
In Review

2. Problem Statement

Users currently cannot see which customers are at risk of churning due to failed payments. They have to check Stripe manually or wait for customer complaints.

3. User Stories

As a SaaS founder,
I want to see a dashboard of all customers with failed payments,
so that I can take action before they churn.
As a revenue manager,
I want to filter customers by retry status and amount at risk,
so that I can prioritize recovery efforts.

4. Acceptance Criteria

✅ Dashboard shows all customers with failed payments in the last 30 days
✅ Users can filter by: retry status, amount, days since failure
✅ Clicking a customer shows full payment history
✅ Export to CSV available
✅ Updates in real-time via WebSocket

5. Success Metrics

• 80% of users check the dashboard weekly
• Time to identify at-risk customers: < 30 seconds
• Feature NPS: > 40
