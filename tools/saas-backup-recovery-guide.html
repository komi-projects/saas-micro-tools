💾 Backup & Recovery

Protect your data. Sleep at night.

The 3-2-1 Backup Rule

3 copies of your data, on 2 different media, with 1 offsite. This is the gold standard for data protection. In SaaS, it translates to: primary database, replica/snapshot, and offsite/cloud backup.

Why? Because hardware fails, humans make mistakes, and disasters happen. A backup you can't restore is worthless. Test your backups monthly.

Database Backups

🔄 Continuous Replication
Streaming replication to a standby server. Near-zero RPO (Recovery Point Objective). Used for HA, not primary backup. If primary corrupts data, replica gets corrupted too. Always pair with point-in-time backups.
📸 Snapshots
Point-in-time disk snapshots (AWS EBS, GCP Persistent Disk). Fast to create, fast to restore. Good for disaster recovery. Take daily, retain 7-30 days. Not a substitute for logical backups (can't restore individual rows easily).
💾 Logical Backups (SQL Dumps)
Export database as SQL (pg_dump, mysqldump). Can restore individual tables or rows. Slower to create and restore than snapshots. Essential for granular recovery. Run daily, retain 30 days locally, 90 days in cloud storage.
☁️ Offsite/Cloud Backups
Upload backups to separate cloud provider (AWS S3, GCS, Backblaze B2). Cross-region replication for maximum resilience. If your primary cloud fails, backups are in a different provider. Retain 90 days minimum, 1 year for compliance.

Backup Strategies by Data Type

Transactional Data (Database):

• Full backup: Daily at low-traffic hours (2-4 AM) • Incremental/wal archiving: Continuous, for point-in-time recovery • Retention: 7 daily, 4 weekly, 12 monthly • Testing: Restore to staging environment weekly • Encryption: Encrypt at rest and in transit

File Storage (S3, uploads):

• Versioning: Enable object versioning on S3 • Cross-region replication: Replicate to different region • Lifecycle policies: Move old versions to cheaper storage (Glacier) • Integrity: Checksums (SHA-256) to detect corruption

Application Code:

• Git repository: Already distributed (every clone is a backup) • Multiple remotes: Push to GitHub AND GitLab AND self-hosted • Release artifacts: Store Docker images, build artifacts in registry with versioning • Infrastructure as code: Terraform/CloudFormation configs in version control

Configuration & Secrets:

• Environment configs: Version controlled (not secrets) • Secrets: Encrypted backup of secret manager (HashiCorp Vault, AWS Secrets Manager) • API keys: Documented with rotation procedures • SSL certificates: Automated renewal (Let's Encrypt) + backup of private keys

Recovery Objectives

RPO (Recovery Point Objective): Maximum acceptable data loss. How much data can you afford to lose? RPO = 1 hour means you can lose up to 1 hour of data. Set by business requirements, not technical convenience.

RTO (Recovery Time Objective): Maximum acceptable downtime. How long can you be down? RTO = 4 hours means service must be restored within 4 hours. Again, business-driven.

Common SaaS targets:

• Tier 1 (core database): RPO < 1 hour, RTO < 4 hours • Tier 2 (file storage): RPO < 24 hours, RTO < 8 hours • Tier 3 (analytics): RPO < 24 hours, RTO < 24 hours • Tier 4 (marketing site): RPO < 72 hours, RTO < 4 hours

Testing Recovery

Untested backups are Schrödinger's backups — they simultaneously exist and don't exist until you try to restore. Test monthly:

1. Restore database to isolated staging environment 2. Verify data integrity (row counts, checksums, spot checks) 3. Verify application works with restored data 4. Document recovery time 5. Fix any issues found 6. Update runbooks with actual recovery steps

Chaos engineering: Simulate failures. Kill a database node during business hours (in staging!). Trigger a restore from backup. Does your team know what to do? Does the process work? Fix gaps before real disasters.

Backup Tools

Managed: AWS RDS automated backups, GCP Cloud SQL backups, Azure Database backups. Set and forget, but verify they work.

Open source: pgBackRest (PostgreSQL), Percona XtraBackup (MySQL), Restic (universal, encrypts and deduplicates), BorgBackup (deduplication, compression).

SaaS: Rewind (Shopify, QuickBooks), Backupify (Google Workspace, Salesforce), OwnBackup (Salesforce). Good for third-party SaaS you depend on.
