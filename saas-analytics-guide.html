Build analytics that drive growth, not just reports.

Analytics Maturity Model

Level 0 — No Analytics: Flying blind. No event tracking. Decisions based on gut feeling and anecdote. Common in pre-launch and very early startups. Risky but sometimes necessary when speed matters more than precision.

Level 1 — Vanity Metrics: Tracking page views, signups, downloads. Dashboards look impressive but don't drive action. "We had 10,000 visitors this month!" So what? Did they convert? Retain? Pay? Most startups plateau here because it feels like progress.

Level 2 — Funnel Tracking: Measuring conversion through key steps. Signup → Activation → Paid → Retained. Can identify where users drop off. "40% of signups never connect their first integration." Now you have a target for improvement.

Level 3 — Cohort Analysis: Tracking groups of users over time. January signups vs. February signups. Did onboarding improvements in February improve March retention? This is where you start understanding causality, not just correlation.

Level 4 — Segmentation: Breaking users into meaningful groups and comparing behavior. Enterprise vs. SMB. Mobile vs. desktop. Organic vs. paid. Different segments have different needs, behaviors, and value. One-size-fits-all metrics hide crucial differences.

Level 5 — Predictive: Using data to predict future outcomes. Churn prediction, LTV forecasting, expansion likelihood. Requires significant data volume and data science resources. Most SaaS companies never reach this level, and that's okay — Levels 3-4 drive most of the value.

Core SaaS Metrics to Track

Acquisition
How users find you • Website visitors by channel (organic, paid, referral, direct) • Signup rate (visitors → signups) • Cost per acquisition (CPA) by channel • Signup source attribution (which campaign, which ad, which landing page) Track daily and weekly. Channel mix changes over time.
Activation
Users experiencing core value • Activation rate (signups → first key action) • Time to first value (how long from signup to activation) • Setup completion rate (e.g., connected integration, invited team, created first project) • Feature adoption (which features do activated users use first) This is the most important funnel stage. Low activation = everything downstream suffers.
Revenue
Money metrics • MRR (Monthly Recurring Revenue) — the north star • ARR (Annual Recurring Revenue) — MRR × 12 • ARPU (Average Revenue Per User) — MRR / paying customers • Expansion MRR — revenue from existing customers upgrading • Contraction MRR — revenue lost from downgrades • Net Revenue Retention (NRR) — (starting + expansion - contraction - churn) / starting Track MRR daily. Everything else weekly or monthly.
Retention
Users coming back • D1, D7, D30 retention (what % of day-0 users are active on day 1, 7, 30) • Cohort retention curves (track each signup cohort over time) • Feature retention (users who use feature X retain Y% better) • Subscription retention (what % renew at period end) Retention is the true product-market fit indicator. If users don't retain, nothing else matters.
Engagement
How users interact with the product • DAU/MAU ratio (daily active / monthly active — proxy for stickiness) • Session frequency (how often users return) • Session duration (time spent per session) • Core action frequency (how often users do the main thing your product does) • Feature usage heatmap (which features are used, which are ignored) Engagement predicts retention. Retention predicts revenue.
Churn & Health
Warning signs and losses • Logo churn rate (% of customers who cancel) • Revenue churn rate (% of MRR lost to cancellations) • Net churn (revenue churn minus expansion) • Customer health score (composite of usage, engagement, support tickets, payment history) • Time-to-churn prediction (which users are likely to cancel soon) Track monthly. React immediately to spikes.

Analytics Tools for SaaS

Product Analytics (Event Tracking):

• Amplitude: The gold standard for product analytics. Cohort analysis, funnel analysis, user paths. Free tier generous. Best for: teams that take analytics seriously. • Mixpanel: Strong competitor to Amplitude. Slightly better UI, slightly fewer advanced features. Great for marketing + product combined. • Heap: Auto-captures all events retroactively. No need to define events upfront. Great for teams that don't know what to track yet. Downsides: data volume costs, noise from auto-capture. • PostHog: Open-source product analytics. Self-host or cloud. Feature flags, session replay, experiments. Great for privacy-conscious and cost-conscious teams. • June: Product analytics for B2B SaaS. Built on top of Segment. Focused on SaaS-specific metrics out of the box. • Google Analytics 4: Free but not designed for SaaS. Good for web traffic, poor for product events and retention. Use alongside product analytics, not instead of.

Data Warehouse / BI:

• Snowflake / BigQuery / Redshift: Central data warehouse. All data sources feed in. SQL-based analysis. Power for data teams. • Metabase: Open-source BI. Non-technical users can build dashboards. Great for democratizing data access. • Looker: Enterprise BI. Powerful but expensive and requires dedicated admin. • Mode: Analytics for analysts. SQL + Python/R notebooks. Great for deep analysis.

Session Replay:

• FullStory: See exactly how users interact with your product. Frustration signals, rage clicks, dead clicks. Pricey but invaluable for UX research. • LogRocket: Session replay + performance monitoring + error tracking. Good for engineering teams. • Hotjar: Heatmaps, session recordings, surveys. Cheaper but less sophisticated than FullStory. Great for landing pages and marketing sites. • PostHog: Includes session replay. Open source and free tier available.

Reverse ETL (Data Activation):

• Hightouch / Census: Sync warehouse data back to tools. E.g., sync "high churn risk" scores from warehouse to Intercom for automated outreach. Closing the loop between analytics and action.

Event Tracking Strategy

Events are the atomic unit of product analytics. Every user action is an event with properties. Good event taxonomy is the foundation of useful analytics.

Naming conventions:

• Object + Action: "Project Created", "User Invited", "Report Exported" • Past tense verbs: "Signed Up" not "Sign Up" • Consistent casing: Pick snake_case or camelCase and stick to it • No property in the event name: "Purchase Completed" with property { amount: 99 } not "Purchase 99 Completed"

Core events every SaaS should track:

1. Signed Up — Account creation (channel, plan, referral source) 2. Onboarding Started — First login or tutorial begin 3. Core Action Done — The "aha moment" (first integration connected, first project created, first email sent — whatever your product's core value is) 4. Upgraded — Free → Paid conversion (plan, amount, payment method) 5. Feature Used — Key feature engagement (feature name, frequency) 6. Invited Team Member — Viral loop activation (role, team size after invite) 7. Renewed — Subscription continuation (plan, amount, period) 8. Churned — Cancellation (reason, plan at cancellation, tenure)

Event properties to include:

• User properties: plan, company size, signup date, acquisition channel • Session properties: device, browser, OS, screen size • Action properties: the specific thing acted upon (project ID, feature name, amount) • Context properties: A/B test variant, feature flag status, page URL

Super properties (track on every event):

• User ID • Account/Company ID (for B2B) • Current plan • Signup date • Acquisition channel • A/B test assignments These let you segment any analysis by these dimensions without joining tables.

Analytics Team Structure

Early stage (1-10 people):

• No dedicated analytics person. Product manager or founder owns metrics. • Use simple tools (PostHog, Amplitude free tier, Metabase) • Focus on 3-5 core metrics. Don't overcomplicate. • Weekly metrics review in team meeting.

Growth stage (10-50 people):

• Hire first data analyst or data engineer. • Implement proper event taxonomy and tracking plan. • Build dashboards for each team (product, marketing, success). • Start cohort analysis and funnel optimization. • Run first A/B tests with statistical rigor.

Scale stage (50+ people):

• Data team: data engineers, analysts, scientists. • Data warehouse as single source of truth. • Self-serve analytics (any team member can answer their own questions). • Predictive models (churn prediction, LTV scoring, expansion likelihood). • Data governance and privacy compliance.

Analytics Mistakes to Avoid

❌ Tracking everything: More data ≠ better decisions. Track events that answer specific questions. "We might need this someday" is not a reason to track an event. ❌ Never looking at the data: Beautiful dashboards nobody checks are waste. Set up alerts for metric changes. Review weekly. Act on insights. ❌ Vanity metrics as KPIs: Page views, downloads, registered users. These go up when you buy ads. Track downstream metrics (activation, retention, revenue) that reflect real value. ❌ Siloed analytics: Marketing tracks leads in HubSpot. Product tracks events in Amplitude. Finance tracks revenue in Stripe. Nobody can answer "Which acquisition channel drives the most valuable customers?" Centralize in a warehouse. ❌ Not tracking from day 1: Retroactive tracking is impossible. Instrument signup, activation, and revenue events before you need them. You will need them. ❌ Changing event names: "User Created" → "Account Created" → "Signup Completed" breaks historical analysis. Pick names and stick to them. Version events if you must change them. ❌ Mixing environments: Development and test data polluting production analytics. Use separate projects or strict environment filtering.

Analytics Checklist

☐ Event taxonomy documented and shared ☐ Core funnel events tracked (signup → activation → paid → retained) ☐ User properties included (plan, channel, company size) ☐ Super properties on every event ☐ Dashboard for each team (product, marketing, success, exec) ☐ Weekly metrics review scheduled ☐ Alerts set for metric anomalies ☐ Cohort analysis configured ␗ A/B test results tracked and documented ☐ Session replay for critical flows ☐ Data warehouse as single source of truth (at scale) ☐ Privacy compliance (GDPR, CCPA) for tracking ☐ Regular data quality audits
