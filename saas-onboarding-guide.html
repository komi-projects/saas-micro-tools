🚀 Onboarding

First impression is everything. Make it count.

Why Onboarding Matters

40-60% of users who sign up for a SaaS product never return after the first visit. Not because the product is bad, but because they never experienced value. Onboarding is the bridge between "signed up" and "actively using."

The goal of onboarding isn't to teach every feature. It's to get the user to their first moment of value — the "aha moment" — as quickly as possible. Slack's aha moment is sending your first message. Dropbox's is syncing your first file. Canva's is creating your first design. Find yours, then optimize the path to it.

Onboarding Principles

1. Reduce friction
Every field, every click, every decision is a chance to drop off. Remove everything non-essential from the signup flow. Email + password + one context question. That's it. Collect the rest later.
2. Guide, don't explain
Don't tell users what your product does. Show them. Empty states should demonstrate the end state. "Your dashboard will look like this" with a screenshot is better than "Add a widget to get started."
3. Personalize the path
Ask one qualifying question during signup: role, use case, or goal. Then customize the onboarding flow. A marketer sees different features than a developer. Context doubles activation rates.
4. Progress, not perfection
Onboarding isn't a one-time event. It's a 30-day journey. Break it into milestones: Day 1 (setup), Day 3 (first action), Day 7 (habit formed), Day 30 (value realized). Celebrate each step.
5. Human backup
Self-serve onboarding is scalable but impersonal. Offer human help at key moments: enterprise gets a call, mid-market gets chat, SMB gets email. The option to talk to a human reduces anxiety.

Onboarding Patterns

Progressive profiling:

Don't ask for everything upfront. Collect data over time as the user engages. First signup: email only. After first use: company name. After first week: team size. After first month: integration needs. Each request is contextual and justified.

Checklists:

Visual progress bars with 3-5 setup tasks. "Complete your profile" → "Connect your first integration" → "Invite a teammate" → "Create your first project." Checklists increase activation by 20-40% because humans love completing things.

Tooltips and hotspots:

Use sparingly. 50 tooltips on first login = overwhelm. 3 well-placed ones that highlight the core workflow = helpful. Never use tooltips for anything the user can figure out themselves. Reserve them for non-obvious but critical features.

Empty states:

Never show a blank screen. Empty states are onboarding opportunities. "You have no reports yet. Create your first report in 2 minutes." Include a CTA, a preview image, and social proof ("Join 10,000+ teams using reports").

Email sequences:

Day 0: Welcome + quick-start guide. Day 1: Tips for first action. Day 3: Best practices. Day 7: Case study. Day 14: Check-in offer. Day 30: Upgrade prompt (if on free trial). Each email has one clear CTA. Unsubscribe is always available.

In-app guidance:

Contextual help based on what the user is doing. "Looks like you're importing data. Here's how to format your CSV." Not generic help docs — help at the moment of need.

Time to First Value (TTFV)

TTFV is the time from signup to the user's first meaningful outcome. The shorter, the better:

• PLG tools: TTFV < 5 minutes. User should get value without any setup. • Team collaboration: TTFV < 1 hour. Invite teammates, see first collaboration. • Complex B2B: TTFV < 1 day. After setup, configuration, and data import. • Enterprise: TTFV < 1 week. Requires implementation, training, integration.

Measure TTFV for each user segment. If marketers have TTFV of 2 hours but developers have 2 days, your developer onboarding is broken.

Reducing TTFV:

• Pre-fill data (import from CSV, connect existing tools) • Templates (don't start from scratch) • Default configurations (smart defaults based on role/industry) • Guided setup wizards (step-by-step, not free-form) • Skip optional setup (let users explore, come back to advanced config later)

Activation Metrics

Activation = the user has experienced core value. Define activation events for your product:

• Slack: Sent 2,000+ messages (not just signed up) • Dropbox: Uploaded 1+ file and synced to 1+ device • HubSpot: Created a contact, sent an email, and viewed a report • Stripe: Processed first payment • Notion: Created 3+ pages and invited 1+ teammate

Your activation event should be: specific, measurable, and correlated with long-term retention. If users who do X are 3x more likely to be active at 30 days, X is your activation event.

Activation rate: % of new signups who reach activation within 7 days. Target: 40-60% for PLG, 20-40% for complex B2B.

Onboarding by Segment

Self-serve (SMB/individual):

• No human touch required • Product-led onboarding with tooltips, checklists, and emails • Free trial or freemium to reduce friction • Automated upgrade prompts based on usage triggers • Support available but not proactive

Sales-assisted (mid-market):

• Self-serve signup + optional sales consultation • Automated onboarding + 1:1 call for complex setups • CSM assigned post-purchase • Hybrid: product drives activation, human drives expansion

Enterprise (high-touch):

• Dedicated implementation team • Custom onboarding plan with milestones • Training sessions (live + recorded) • Regular check-ins (weekly during implementation, monthly after) • Success metrics defined in contract • Executive business reviews quarterly

Onboarding Mistakes

❌ Feature tour overload: Showing every feature in a 10-step tour. Users forget 90% of it. Focus on the 3 things they need to do first.

❌ Blank slate terror: Empty dashboard with no guidance. Users don't know where to start. Always provide templates, samples, or demos.

❌ One-size-fits-all: Same onboarding for every user type. A developer and a marketer need completely different first experiences.

❌ Set it and forget it: Onboarding built once, never updated. Products evolve. Onboarding must evolve with them. Review quarterly.

❌ No feedback loop: Not measuring where users drop off. Use funnel analytics (Amplitude, Mixpanel, Heap) to identify and fix drop-off points.

❌ All product, no human: Even in PLG, some users need help. Offer human support at critical moments — it converts unsure users into advocates.
